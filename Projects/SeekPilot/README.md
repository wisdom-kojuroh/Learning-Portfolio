# SeekPilot

**"Transforming Ambiguous Thoughts into Logical Sequences."**

SeekPilotは、生成AIが出力する「箇条書きのテキスト」を、関係者（アクター）と
その所作がひと目でわかる「構造化されたグリッド図（スイムレーン図）」へと昇華させる、次世代のコンテキスト認識型AIアシスタントです。

---

## 🚀 Concept: 曖昧な思考を、確実な構造へ

現在のAIは情報を「テキスト」として出力することには長けていますが、複数の登場人物が複雑に絡み合う「プロセス」を可視化するには至っていません。
SeekPilotは、AIが生成した中間言語を独自のエンジンで解析し、論理的な正確性と視覚的な直感性を兼ね備えたUIを提供します。

---

## 💡 Use Cases: 活用イメージ

### 1. 法的手続き・コンプライアンスの可視化

- **宅建業法における営業保証金供託フロー**
    - 「業者」「供託所」「免許権者」の3レーンを自動生成。第25条に基づく厳格な法的順序（供託→届出→営業開始）をミスなく可視化します。

### 2. システムアーキテクチャ・データ転送

- **マイクロサービス間の認証フロー**
    - クライアント、API Gateway、Auth Service間のデータの受渡しを即座に図解。設計時の認識合わせを加速させます。

### 3. 社会・公共プロセスの構造化

- **立法プロセス（例：議員定数削減フロー）**
    - 政党、内閣、衆参両院といった多数のアクターが関わる政治的トピックを構造化。専門知識がなくても社会の仕組みを直感的に理解可能にします。

## 🛠 Technical Stack

- **Language:** Python (自然言語処理・ロジック実装)
- **Backend:** FastAPI (AI出力の構造化パースエンジン)
- **Frontend:** React / TypeScript (動的なグリッドレンダリング)
- **AI Engine:** Gemini API (コンテキスト抽出・中間言語生成)
- **Interface:** Chrome Extension (ブラウザ内コンテキスト認識)

---

## 🏗 System Architecture

1. **Input:** ブラウザ上のテキストまたはユーザーの曖昧な指示。
2. **Brain (LLM):** 指示を解析し、独自の記号ベースの中間言語（JSON）へ変換。
3. **Engine (SeekPilot):** 中間言語をパースし、座標・アクター・アクションの構造体を作成。
4. **Output:** スイムレーン形式のグリッド図をユーザーインターフェースにレンダリング。

---

## 🗺 Roadmap

- **Phase 1: MVP (Core Engine)** - テキストからJSONへの変換、およびシンプルなグリッド描画の実装。
- **Phase 2: AI Integration** - Gemini APIを用いた自動抽出プロンプトの最適化。
- **Phase 3: Browser Assistant** - Chrome拡張機能としてブラウザ上の情報をリアルタイム図解。
- **Phase 4: Multi-Actor Support** - 政治プロセスのような多重アクター環境への最適化。

## 🖋 Development Background

宅地建物取引業法などの複雑な法律を学ぶ際、条文を読み解くだけでは「誰が・いつ・どこに」という時系列の把握に限界を感じたことが開発のきっかけです。
「AIが答えを出す」時代から、**「AIが理解を構造化する」**時代へ。SeekPilotはその架け橋となるツールを目指しています。
